# 风险管理

### 引言

金星协议 致力于为用户提供最高级别的安全性和风险管理。本文档的这一部分概述了我们全面的风险管理方法，重点介绍了我们的持续监控实践、增强安全性的合作伙伴关系、实时警报系统以及持续的风险评估。

### 部署前多次审计

* 在部署之前，金星协议 会接受由行业领先的安全公司进行的全面审计。这些审计结果已公开，并已全面实施，从而确保了稳健安全的框架。

* 点击此处访问我们的详细审计报告：[安全与审计](https://docs-v4.venus.io/links/security-and-audits)

### 持续链上监控

* 金星协议 已与区块链安全和监控领域的领导者 Chaos Labs 建立合作伙伴关系。此次合作使我们能够使用先进的工具进行持续的链上监控，从而显著增强我们的风险控制和缓解策略。

* 浏览我们的监控仪表盘，获取实时洞察：[Chaos Labs 风险概览](https://community.chaoslabs.xyz/venus/risk/overview)。

### 实时警报系统

* 为了进一步加强风险管理，我们已通过 电报群 部署了实时警报系统。该系统可针对市场高利用率、巨鲸资金的重大变动以及借贷/供应上限使用等关键事件提供即时通知。

* 加入我们的警报频道，随时了解最新动态：[Chaos Labs Telegram 警报](https://community.chaoslabs.xyz/venus/risk/alerts)。

### 动态风险管理

* 我们与 Chaos Labs 合作，持续分析市场状况，并根据市场状况调整所有资金池的风险参数，定期或随时进行调整以降低风险。这些调整基于全面的数据分析，是维护协议稳定性和安全性的关键所在。

### 前端

我们已实施多项安全措施，以降低部署在 [https://app.venus.io](https://app.venus.io) 的前端应用程序的安全风险。

* 金星协议 用户界面部署在 AWS 基础设施上。此外，我们使用 Cloudflare 分发 Web 应用程序。AWS 和 Cloudflare 中的相关安全服务均已正确配置，以检测和降低安全风险。

* 我们配置了 SPF 和 DKIM，以保护从 @venus.io 帐户发送的每封电子邮件。

* 由 AWS 生成的 Web 证书已设置为自动续订。

* 金星 域名服务器托管在 Cloudflare 上，Cloudflare 是我们的 DNS 提供商。

* 金星协议 用户界面使用 [Chainalysis](https://www.chainalysis.com) 来评估 web3 地址的风险状况。如果风险被判定为高风险，则连接将被禁止，从而阻止用户界面与涉及该风险地址的任何合约进行交互。

只有特权用户才有权将 GitHub 上的开源代码（https://github.com/VenusProtocol/venus-protocol-interface）部署到 app.venus.io 域名。此外，也只有特权用户才有权合并 GitHub 代码库中的拉取请求。

### 结论

金星协议 的风险管理实践旨在为用户提供安全稳定的环境。我们致力于持续监控、实时警报和动态风险调整，确保始终处于区块链安全领域的前沿。我们鼓励社区成员探索我们提供的资源，以更深入地了解我们的风险管理协议。
